Funciones Proposicionales#

Concepto

Una función proposicional es verdadera o falsa dependiendo del valor de su parámetro.

  • Escribe una función que indique si un número es estrictamente mayor que \(\pi\)

  • Escribe una función que reciba dos números \(a\) y \(b\) e indique si \(a\) es estrictamente mayor que \(b\).

import math

def Mayor_que_pi(x):
  return x > math.pi

def Comparar_numeros(a,b):
  return a>b

Conectivos Lógicos#

Concepto

Para unir dos proposiciones atómicas y formar una proposición molecular, podemos usar conectivos lógicos. En Python disponemos de tres principales:

  • not → negación

  • and → conjunción

  • or → disyunción

A continuación podemos escribir algunas proposiciones moleculares utilizando estos conectivos.

not (3>6)

True

x=19
(x<=20) and (x>16)

True

\(P \Rightarrow Q \equiv \neg P \lor Q\)

def Implicacion(P,Q):
  return (not P) or Q

P=False
Q=True

Implicacion(P,Q)

True

En Python, los conectivos lógicos tienen un orden de prioridad:

  1. not (mayor prioridad)

  2. and

  3. or (menor prioridad)

Así, la expresión:

not (x == 3) and (x < 4)

se evalúa como:

(not (x == 3)) and (x < 4)

Es decir, primero se niega (x == 3), y luego se verifica la conjunción con (x < 4). Para evitar ambigüedades, siempre es recomendable usar paréntesis de forma explícita.

Equivalencias Lógicas#

Concepto

Dos proposiciones son equivalentes si tienen el mismo valor de verdad para cualquier combinación de valores de verdad de las proposiciones que las componen.

Verificaremos las siguientes equivalencias:

  1. Ley de De Morgan: \(\neg(P \wedge Q) \equiv \neg P \vee \neg Q\)

  2. Distributividad: \(P \wedge (Q \vee R) \equiv (P \wedge Q) \vee (P \wedge R)\)

  3. Negación de la implicación: \(\neg(P \Rightarrow Q) \equiv P \wedge \neg Q\)

  4. Contrapuesta: \(P \Rightarrow Q \equiv \neg Q \Rightarrow \neg P\)

  5. La implicación no es conmutativa: \(P \Rightarrow Q \not\equiv Q \Rightarrow P\)

Nota: la implicación (\(\Rightarrow\)) no existe como tal en Python. Usaremos la equivalencia que podemos definir como funcion: \(P \Rightarrow Q \equiv \neg P \vee Q\)

Leyes Válidas de Inferencia#

Concepto

Una ley de inferencia nos permite concluir una proposición a partir de otras proposiciones dadas. Estas reglas son siempre válidas, es decir, son tautologías: su valor de verdad es True para cualquier combinación de valores de verdad de las proposiciones que las componen.

Verificaremos las siguientes leyes:

  1. Modus ponens: \((P \land (P \Rightarrow Q)) \Rightarrow Q\)

  2. Modus tollendo ponens: \(((\neg P) \land (P \vee Q)) \Rightarrow Q\)

  3. Modus tollendo tollens: \(((\neg Q) \land (P \Rightarrow Q)) \Rightarrow (\neg P)\)

  4. Silogismo: \(((P \Rightarrow Q) \land (Q \Rightarrow R)) \Rightarrow (P \Rightarrow R)\)

print("P\tQ\tR\tSilogismo")

for P in [True, False]:
  for Q in [True, False]:
    for R in [True, False]:
      A=Implicacion(P,Q)
      B=Implicacion(Q,R)
      C=Implicacion(P,R)
      Silogismo=Implicacion(A and B, C)
      print(P, "\t", Q, "\t", R, "\t", Silogismo )

Conjuntos#

Concepto

Un conjunto es una colección de elementos sin orden y sin repetición.

Conjunto de letras \(A = \{"a", "b", "c"\}\)

Conjunto de números \(B = \{1, 2, 3, 4\}\)

C={4, 6, 8, 10, 10}

print(C)
print(D)

Nota: estas definiciones son dadas por extensión, es decir, indicando explícitamente todos los elementos que pertenecen al conjunto.

¿Qué pasa si repetimos un elemento? Por ejemplo:

C = {1, 2, 2, 3, 3, 3}
print(C)

C = {1, 2, 2, 3, 3, 3}
print(C)
